## 1. Nature

Pacioli is a business system:

* Life-cycle model:

    * Agile development
        * Extreme Programming
        * Scrum
        * timebox development
    * Evolutionary prototyping

* Planning and management:
    * Incremental project planning
    * As-needed test and QA planning
    * Informal change control

* Requirements:
    * Informal requirements specification

* Design:
    * Design and coding are combined

## 2. Objective

The objective of Pacioli is to provide  quantitative information for financial decisions.

## 3. Specific Functional Requirements

* Are all the inputs to the system specified, including their source, accuracy, range of values, and frequency?

* Are all the outputs from the system specified, including their destination, accuracy, range of values, frequency, and format?

* Are all output formats specified for Web pages and reports?

* Is the API specified?

* Are all the tasks the user wants to perform specified?

* Is the data used in each task and the data resulting from each task specified?

## 4. Specific Nonfunctional (Quality) Requirements

* Is the level of security specified?
* Is the reliability specified, including the consequences of software failure, the vital information that needs to be protected from failure, and the strategy for error detection and recovery?
